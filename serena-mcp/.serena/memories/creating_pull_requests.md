# Creating Pull Requests

* Make sure the instructions in `CONTRIBUTING.md` regarding the scope of PRs
  are followed.
* For changes that add features or fix problems, 
  add an entry to `CHANGELOG.md` which *concisely* describes the change.
  Note that details/background information on the change should be in the commit 
  message, not in the changelog.
  Make sure to follow the existing format and to add your change to the 
  appropriate section.
